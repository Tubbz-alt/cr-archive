468   // Request the collector enter the indicated concurrent phase, and
469   // wait until it does so.  Supports WhiteBox testing.  Only one
470   // request may be active at a time.  Phases are designated by name;
471   // the set of names and their meaning is GC-specific.  Once the
472   // requested phase has been reached, the collector will attempt to
473   // avoid transitioning to a new phase until a new request is made.
474   // [Note: A collector might not be able to remain in a given phase.
475   // For example, a full collection might cancel an in-progress
476   // concurrent collection.]
477   //
478   // Returns true when the phase is reached.  Returns false for an
479   // unknown phase.  The default implementation returns false.
480   virtual bool request_concurrent_phase(const char* phase);

486   // if the GC can pause and resume concurrent work (e.g. G1
487   // concurrent marking) for an intermittent non-GC safepoint.
488   // If this method returns NULL, SafepointSynchronize will
