 27 MSG_Help=Usage: jpackage <options>\n\
 28 \n\
 29 Sample usages:\n\
 30 --------------\n\
 31 \    Generate an application package suitable for the host system:\n\
 32 \        For a modular application:\n\
 33 \            jpackage -n name -p modulePath -m moduleName/className\n\
 34 \        For a non-modular application:\n\
 35 \            jpackage -i inputDir -n name \\\n\
 36 \                --main-class className --main-jar myJar.jar\n\
 37 \        From a pre-built application image:\n\
 38 \            jpackage -n name --app-image appImageDir\n\
 39 \    Generate an application image:\n\
 40 \        For a modular application:\n\
 41 \            jpackage --type app-image -n name -p modulePath \\\n\
 42 \                -m moduleName/className\n\
 43 \        For a non-modular application:\n\
 44 \            jpackage --type app-image -i inputDir -n name \\\n\
 45 \                --main-class className --main-jar myJar.jar\n\
 46 \        To provide your own options to jlink, run jlink separately:\n\
 47 \            jlink --output appRuntimeImage -p modulePath -m moduleName \\\n\
 48 \                --no-header-files [<additional jlink options>...]\n\
 49 \            jpackage --type app-image -n name \\\n\
 50 \                -m moduleName/className --runtime-image appRuntimeImage\n\
 51 \    Generate a Java runtime package:\n\
 52 \        jpackage -n name --runtime-image <runtime-image>\n\
 53 \n\
 54 Generic Options:\n\
 55 \  @<filename> \n\
 56 \          Read options and/or mode from a file \n\
 57 \          This option can be used multiple times.\n\
 58 \  --type -t <type> \n\
 59 \          The type of package to create\n\
 60 \          Valid values are: {1} \n\
 61 \          If this option is not specified a platform dependent\n\
 62 \          default type will be created.\n\
 63 \  --app-version <version>\n\
 64 \          Version of the application and/or package\n\
 65 \  --copyright <copyright string>\n\
 66 \          Copyright for the application\n\
 67 \  --description <description string>\n\
 68 \          Description of the application\n\
 69 \  --help -h \n\
 70 \          Print the usage text with a list and description of each valid\n\
 71 \          option for the current platform to the output stream, and exit\n\
 72 \  --name -n <name>\n\
 73 \          Name of the application and/or package\n\
 74 \  --dest -d <destination path>\n\
 75 \          Path where generated output file is placed\n\
 76 \          Defaults to the current working directory.\n\
 77 \          (absolute path or relative to the current directory)\n\
 78 \  --temp <file path>\n\
 79 \          Path of a new or empty directory used to create temporary files\n\
 80 \          (absolute path or relative to the current directory)\n\
 81 \          If specified, the temp dir will not be removed upon the task\n\
 82 \          completion and must be removed manually\n\
 83 \          If not specified, a temporary directory will be created and\n\
 84 \          removed upon the task completion.\n\
 85 \  --vendor <vendor string>\n\
 86 \          Vendor of the application\n\
 87 \  --verbose\n\
 88 \          Enables verbose output\n\
 89 \  --version\n\
 90 \          Print the product version to the output stream and exit\n\
 91 \n\
 92 \Options for creating the runtime image:\n\
 93 \  --add-modules <module name>[,<module name>...]\n\
 94 \          A comma (",") separated list of modules to add.\n\
 95 \          This module list, along with the main module (if specified)\n\
 96 \          will be passed to jlink as the --add-module argument.\n\
 97 \          if not specified, either just the main module (if --module is\n\
 98 \          specified), or the default set of modules (if --main-jar is \n\
 99 \          specified) are used.\n\
100 \          This option can be used multiple times.\n\
101 \  --module-path -p <module path>...\n\
102 \          A {0} separated list of paths\n\
103 \          Each path is either a directory of modules or the path to a\n\
104 \          modular jar.\n\
105 \          (each path is absolute or relative to the current directory)\n\
106 \          This option can be used multiple times.\n\
107 \  --bind-services \n\
108 \          Pass on --bind-services option to jlink (which will link in \n\
109 \          service provider modules and their dependences) \n\
110 \          This option is deprecated. Use "--jlink-options" option instead. \n\
111 \  --jlink-options <jlink options> \n\
112 \          A space separated list of options to pass to jlink \n\
113 \          If not specified, defaults to "--strip-native-commands \n\
114 \          --strip-debug --no-man-pages --no-header-files" \n\
115 \          This option can be used multiple times.\n\
116 \  --runtime-image <file path>\n\
117 \          Path of the predefined runtime image that will be copied into\n\
118 \          the application image\n\
119 \          (absolute path or relative to the current directory)\n\
120 \          If --runtime-image is not specified, jpackage will run jlink to\n\
121 \          create the runtime image using options:\n\
122 \          --strip-debug, --no-header-files, --no-man-pages, and\n\
123 \          --strip-native-commands.\n\
124 \n\
125 \Options for creating the application image:\n\
126 \  --icon <icon file path>\n\
127 \          Path of the icon of the application package\n\
128 \          (absolute path or relative to the current directory)\n\
129 \  --input -i <input path>\n\
130 \          Path of the input directory that contains the files to be packaged\n\
131 \          (absolute path or relative to the current directory)\n\
132 \          All files in the input directory will be packaged into the\n\
133 \          application image.\n\
134 \n\
135 \Options for creating the application launcher(s):\n\
136 \  --add-launcher <launcher name>=<file path>\n\
137 \          Name of launcher, and a path to a Properties file that contains\n\
138 \          a list of key, value pairs\n\
139 \          (absolute path or relative to the current directory)\n\
140 \          The keys "module", "main-jar", "main-class",\n\
141 \          "arguments", "java-options", "app-version", "icon", and\n\
142 \          "win-console" can be used.\n\
143 \          These options are added to, or used to overwrite, the original\n\
144 \          command line options to build an additional alternative launcher.\n\
145 \          The main application launcher will be built from the command line\n\
146 \          options. Additional alternative launchers can be built using\n\
147 \          this option, and this option can be used multiple times to\n\
148 \          build multiple additional launchers. \n\
149 \  --arguments <main class arguments>\n\
150 \          Command line arguments to pass to the main class if no command\n\
151 \          line arguments are given to the launcher\n\
152 \          This option can be used multiple times.\n\
153 \  --java-options <java options>\n\
154 \          Options to pass to the Java runtime\n\
155 \          This option can be used multiple times.\n\
156 \  --main-class <class name>\n\
157 \          Qualified name of the application main class to execute\n\
158 \          This option can only be used if --main-jar is specified.\n\
159 \  --main-jar <main jar file>\n\
160 \          The main JAR of the application; containing the main class\n\
161 \          (specified as a path relative to the input path)\n\
162 \          Either --module or --main-jar option can be specified but not\n\
163 \          both.\n\
164 \  --module -m <module name>[/<main class>]\n\
165 \          The main module (and optionally main class) of the application\n\
166 \          This module must be located on the module path.\n\
167 \          When this option is specified, the main module will be linked\n\
168 \          in the Java runtime image.  Either --module or --main-jar\n\
169 \          option can be specified but not both.\n\
170 {2}\n\
171 \Options for creating the application package:\n\
172 \  --app-image <file path>\n\
173 \          Location of the predefined application image that is used\n\
174 \          to build an installable package\n\
175 \          (absolute path or relative to the current directory)\n\
176 \  --file-associations <file path>\n\
177 \          Path to a Properties file that contains list of key, value pairs\n\
178 \          (absolute path or relative to the current directory)\n\
179 \          The keys "extension", "mime-type", "icon", and "description"\n\
180 \          can be used to describe the association.\n\
181 \          This option can be used multiple times.\n\
182 \  --install-dir <file path>\n\
183 \          {4}\
184 \  --license-file <file path>\n\
185 \          Path to the license file\n\
186 \          (absolute path or relative to the current directory)\n\
187 \  --resource-dir <path>\n\
188 \          Path to override jpackage resources\n\
189 \          Icons, template files, and other resources of jpackage can be\n\
190 \          over-ridden by adding replacement resources to this directory.\n\
191 \          (absolute path or relative to the current directory)\n\
192 \  --runtime-image <file-path>\n\
193 \          Path of the predefined runtime image to install\n\
194 \          (absolute path or relative to the current directory)\n\
195 \          Option is required when creating a runtime package.\n\
