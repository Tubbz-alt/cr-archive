26  * @test
27  * @key stress gc
28  *
29  * @summary converted from VM Testbase gc/gctests/PhantomReference/phantom002.
30  * VM Testbase keywords: [gc, stress, stressopt, nonconcurrent]
31  * VM Testbase readme:
32  * DESCRIPTION
33  *     The test checks that Garbage Collector correctly works with
34  *     PhantomReferences. It also checks that no unexpected exceptions and errors
35  *     are thrown or the JVM is not crashed.
36  *     The test starts a number of threads. Each thread run tests for some time
37  *     or serveral iterations.  See javadoc StressOptions for configuration.
38  *     First of all each thread defines what type to check (there are 11 types
39  *     totally). As soon as the type is defined, a PhantomRefence is created that
40  *     refers to an array of tested type and is registered with in a queue. A
41  *     PhantomRefence for NonbranchyTree and Referent calsses does not refer to
42  *     arrays, but to instances of the classes.
43  *     After that a thread performs next checks for the reference:
44  *         1. The reference is in queue after GC is provoked with
45  *            Algorithms.eatMemory() method (a single thread eats the memory).
46  *         2. reference.get() returns null.
47  *         3. queue.poll() returns the reference that was created.
48  *         4. queue.poll() again returns null.
49  *         5. If the checked type is class (Referent), then it must be finalized,
50  *            since the reference is already enqueued.
51  *         6. reference.clear() does not throw any exception.
52  *     The test extends ThreadedGCTest and implements GarbageProducerAware and
53  *     MemoryStrategyAware interfaces. The corresponding javadoc documentation
54  *     for additional test configuration.
55  *
56  * @library /vmTestbase
57  *          /test/lib
58  * @run driver jdk.test.lib.FileInstaller . .
59  * @run main/othervm gc.gctests.PhantomReference.phantom001.phantom001 -ms high
