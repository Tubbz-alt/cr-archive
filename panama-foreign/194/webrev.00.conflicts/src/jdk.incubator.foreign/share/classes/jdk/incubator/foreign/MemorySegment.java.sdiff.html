317      * Performs a bulk copy from given source segment to this segment. More specifically, the bytes at
318      * offset {@code 0} through {@code src.byteSize() - 1} in the source segment are copied into this segment
319      * at offset {@code 0} through {@code src.byteSize() - 1}.
320      * If the source segment overlaps with this segment, then the copying is performed as if the bytes at
321      * offset {@code 0} through {@code src.byteSize() - 1} in the source segment were first copied into a
322      * temporary segment with size {@code bytes}, and then the contents of the temporary segment were copied into
323      * this segment at offset {@code 0} through {@code src.byteSize() - 1}.
324      * <p>
325      * The result of a bulk copy is unspecified if, in the uncommon case, the source segment and this segment
326      * do not overlap, but refer to overlapping regions of the same backing storage using different addresses.
327      * For example, this may occur if the same file is {@link MemorySegment#mapFromPath mapped} to two segments.
328      *
329      * @param src the source segment.

330      * @throws IndexOutOfBoundsException if {@code src.byteSize() > this.byteSize()}.



331      * @throws IllegalStateException if either the source segment or this segment have been already closed,
332      * or if access occurs from a thread other than the thread owning either segment.
333      * @throws UnsupportedOperationException if either the source segment or this segment do not feature required access modes;
334      * more specifically, {@code src} should feature at least the {@link MemorySegment#READ} access mode,
335      * while this segment should feature at least the {@link MemorySegment#WRITE} access mode.
