30  * VM Testbase readme:
31  * ================================================
32  * DESCRIPTION:
33  *         The test is aimed on the control of:
34  *         Interface:      com.sun.jdi.connect.Connector.Argument
35  *         Method:         public java.lang.String value()
36  *         Assertion:      "Returns the current value of the argument."
37  * ================================================
38  * COMMENTS:
39  *         The test aims on the concrete Sun's JDI
40  *         reference implementations. It uses
41  *         com.sun.jdi.RawCommandLineLaunch connector and its
42  *         "command" argument.
43  *         The test sets up the new "command" argument
44  *         value and then checks that new value remains previously
45  *         after connection establishing with debugee VM and
46  *         after debugee VM finishing.
47  *     Test was fixed according to test bug:
48  *     4778296 TEST_BUG: debuggee VM intemittently hangs after resuming
49  *     - debuggee wrapper is used to handle VMStartEvent
50  * ================================================
51  * WARNING:
52  *         Temporarily the test is prepared only for
53  *         Sparc.Solaris.dt_socket-transport of RawCommandLineLaunch
54  *         connector
55  * ================================================
56  *
57  * @library /vmTestbase
58  *          /test/lib
59  * @run driver jdk.test.lib.FileInstaller . .
60  * @build nsk.jdi.Argument.value.value004
61  *        nsk.jdi.Argument.value.value004a
62  * @run main/othervm PropertyResolvingWrapper
63  *      nsk.jdi.Argument.value.value004
64  *      -verbose
65  *      -arch=${os.family}-${os.simpleArch}
66  *      -waittime=5
67  *      -debugee.vmkind=java
68  *      -transport.address=dynamic
69  *      "-debugee.vmkeys=${test.vm.opts} ${test.java.opts}"
70  */
