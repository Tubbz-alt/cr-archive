 13 <h1>Glossary</h1>

 19 
 20 <dl>
 21 <dt><a name="accepted"><strong>accepted</strong> (by the CCC)</a></dt>
 22 <dd>The stage of the CCC process after "DRAFT", and "PROPOSED".  At this stage
 23     the primary goals are to ensure that the proposed changes are suitable for
 24     the release in a general sense and that the requisite JCK and SQE resources
 25     will be available.</dd>
 26 </dl>
 27 
 28 <dl>
 29 <dt><a name="approved"><strong>approved</strong> (by the CCC)</a></dt>
 30 <dd>The stage of the CCC process after "FINAL".  The CCC has approved the
 31     final version of the request which permits push into the project
 32     forest.</dd> 
 33 </dl>
 34 
 35 <dl>
 36 <dt><a name="changeset"><strong>changeset</strong></a></dt>
 37 <dd>A collection of changes with respect to the current clone of a
 38     repository.</dd>  
 39 </dl>
 40 
 41 <dl>
 42 <dt><a name="devFreeze"><strong>development freeze</strong></a></dt>
 43 <dd>The date by which all planned work should be complete for a particular line
 44     of development.  After a line's development freeze, only exit-criteria bugs
 45     may be fixed in that line.</dd>
 46 </dl>
 47 
 48 <dl>
 49 <dt><a name="forest"><strong>forest</strong></a></dt>
 50 <dd>A collection of Mercurial repositories which can be managed as a set
 51     of nested repositories. The name "Forest" originally came from the
 52     Mercurial "Forest Extension" which can be used with some versions of
 53     Mercurial, but in general is no longer recommended.
 54     The script <code>common/bin/hgforest.sh</code> can be used to apply a
 55     Mercurial <code>hg</code> command to all the repositories in a forest.</dd>
 56 </dl>
 57 
 58 <dl>
 59 <dt><a name="group"><strong>Group</strong></a></dt>
 60 <dd>A collection of <a href="/bylaws#participant">Participants</a> who engage
 61     in open conversation about a common interest. That interest may be in the
 62     creation, enhancement, or maintenance of a specific body of code or it may
 63     lie in other areas, e.g., quality, documentation, or evangelism.  See
 64     the <a href="../groups/">Group overview</a>.</dd>
 65 </dl>
 66 
 67 <dl>
 68 <dt><a name="Mercurial"><strong>Mercurial</strong></a></dt>
 69 <dd>A free, cross-platform, distributed source management tool.  Source bundles
 70     and binary packages for Mercurial are available at <a
 71     href="http://www.selenic.com/mercurial/wiki/index.cgi/Mercurial">http://www.selenic.com/mercurial/wiki/index.cgi</a>.
 72     The main Mercurial documentation is available at <a
 73     href="http://hgbook.red-bean.com/">http://hgbook.red-bean.com</a>.</dd>
 74 </dl>
 75 
 76 <dl>
 77 <dt><a name="project"><strong>Project</strong></a></dt>
 78 <dd>A collaborative effort to produce a specific artifact, which may
 79     be a body of code, or documentation, or some other material.  See the <a 
 80     href="../projects/">Project overview</a>.</dd>
 81 </dl>
 82 
 83 <dl>
 84 <dt><a name="repository"><strong>repository</strong></a></dt>
 85 <dd>A directory tree in the filesystem that Mercurial treats specially.  This
 86     tree contains the source files and their revision history.</dd>
 87 </dl>
 88 
 89 <dl>
 90 <dt><a name="webrev"><strong>webrev</strong></a></dt>
 91 <dd>A tool and its output.  In JDK release forests, the script, 
 92    <a href="http://hg.openjdk.java.net/code-tools/webrev/raw-file/tip/webrev.ksh"><code>webrev.ksh</code></a>, 
 93     examines a forest or repository to generate a set of web-based views of
 94     differences. </dd>
 95 </dl>
