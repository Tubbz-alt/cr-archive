 489      List the jshell tool commands and help subjects\n\n\
 490 /help <command>\n\t\
 491      Display information about the specified command. The slash must be included.\n\t\
 492      Only the first few letters of the command are needed -- if more than one\n\t\
 493      each will be displayed.  Example:  /help /li\n\n\

 499 help.set =\
 500 Set the jshell tool configuration information, including:\n\
 501 the external editor to use, the startup definitions to use, a new feedback mode,\n\
 502 the command prompt, the feedback mode to use, or the format of output.\n\
 503 \n\
 504 /set editor [-wait] <command> <optional-arg>...\n\t\
 505      Specify the command to launch for the /edit command.\n\t\
 506      The <command> is an operating system dependent string\n\n\
 507 /set start <file>\n\t\
 508      The contents of the specified <file> become the default startup snippets and commands\n\n\
 509 /set feedback <mode>\n\t\
 510      Set the feedback mode describing displayed feedback for entered snippets and commands\n\n\
 511 /set mode <mode> [<old-mode>] -command|-quiet|-delete\n\t\
 512      Create or update a user-defined feedback mode, optionally copying from an existing mode\n\n\
 513 /set prompt <mode> "<prompt>" "<continuation-prompt>"\n\t\
 514      Set the displayed prompts for a given feedback mode\n\n\
 515 /set truncation <mode> <length> <selector>...\n\t\
 516      Set the maximum length of a displayed value\n\n\
 517 /set format <mode> <field> "<format>" <selector>...\n\t\
 518      Configure a feedback mode by setting the format of a field when the selector matches\n\n\
 519 /set\n\t\
 520      Show editor, start, and feedback settings as /set commands.\n\t\
 521      To show the settings of any of the above, omit the set value\n\n\
 522 To get more information about one of these forms, use /help with the form specified.\n\
 523 For example:   /help /set format

 838                 "errorline" field only)\n\
 839 The following fields are accessed by the tool to determine the displayed feedback:\n\t\
 840 {display}    == The displayed message for a snippet event\n\t\
 841 {errorline}  == The format of one error line within the "errors" field\n\t\
 842 {pre}        == The feedback prefix (begins command feedback)\n\t\
 843 {post}       == The feedback postfix (ends command feedback)\n\t\
 844 {errorpre}   == The error prefix (begins error feedback)\n\t\
 845 {errorpost}  == The error postfix (ends error feedback)\n\
 846 These fields have default settings (which may be overwritten).\n\
 847 Where <selector> is the context in which the format is applied.\n\
 848 The structure of selector is a hyphen separated list of selector kind lists.\n\
 849 A selector kind list is a comma separated list of values of one selector kind.\n\
 850 A selector matches if each selector kind list matches; A selector kind list\n\
 851 matches if one of the values matches.\n\n\
 852 The case selector kind describes the kind of snippet.  The values are:\n\t\
 853    import     -- import declaration\n\t\
 854    class      -- class declaration\n\t\
 855    interface  -- interface declaration\n\t\
 856    enum       -- enum declaration\n\t\
 857    annotation -- annotation interface declaration\n\t\

 858    method     -- method declaration -- note: {type}==parameter-types\n\t\
 859    vardecl    -- variable declaration without init\n\t\
 860    varinit    -- variable declaration with init\n\t\
 861    expression -- expression -- note: {name}==scratch-variable-name\n\t\
 862    varvalue   -- variable value expression\n\t\
 863    assignment -- assign variable\n\t\
 864    statement  -- statement\n\
 865 The action selector kind describes what happened to the snippet.  The values are:\n\t\
 866    added     -- snippet has been added\n\t\
 867    modified  -- an existing snippet has been modified\n\t\
 868    replaced  -- an existing snippet has been replaced with a new snippet\n\t\
 869    overwrote -- an existing snippet has been overwritten\n\t\
 870    dropped   -- snippet has been dropped\n\t\
 871    used      -- snippet was used when it cannot be\n\
 872 The when-did-it-occur selector kind describes if this is a direct or indirect action.  The values are:\n\t\
 873    primary -- the entered snippet\n\t\
 874    update  -- an update to a dependent snippet\n\
 875 The resolution-state selector kind describes the state of resolution/definition of the snippet.  The values are:\n\t\
 876    ok         -- resolved correctly\n\t\
 877    defined    -- defined despite recoverably unresolved references\n\t\
