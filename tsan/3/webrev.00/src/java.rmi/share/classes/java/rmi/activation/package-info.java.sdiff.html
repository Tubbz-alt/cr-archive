26 /**
27  * Provides support for RMI Object Activation.  A remote
28  * object's reference can be made ``persistent'' and later activated into a
29  * ``live'' object using the RMI activation mechanism.
30  *
31  * <p>Implementations are not required to support the activation
32  * mechanism. If activation is not supported by this implementation,
33  * several specific activation API methods are all required to throw
34  * {@code UnsupportedOperationException}. If activation is supported by this
35  * implementation, these methods must never throw {@code
36  * UnsupportedOperationException}. These methods are denoted by the
37  * presence of an entry for {@code UnsupportedOperationException} in the
38  * <strong>Throws</strong> section of each method's specification.
39  *
40  * @since 1.2
