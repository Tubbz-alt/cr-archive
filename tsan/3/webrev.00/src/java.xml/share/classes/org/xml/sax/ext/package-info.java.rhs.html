26 /**
27  * Provides interfaces to SAX2 facilities that
28  * conformant SAX drivers won't necessarily support.
29  *



30  *
31  * <p>
32  * This package is independent of the SAX2 core, though the functionality
33  * exposed generally needs to be implemented within a parser core.
34  * That independence has several consequences:
35  *
36  * <ul>
37  *
38  * <li>SAX2 drivers are <em>not</em> required to recognize these handlers.
39  * </li>
40  *
41  * <li>You cannot assume that the class files will be present in every SAX2
42  * installation.</li>
43  *
44  * <li>This package may be updated independently of SAX2 (i.e. new
45  * handlers and classes may be added without updating SAX2 itself).</li>
46  *
47  * <li>The new handlers are not implemented by the SAX2
48  * <code>org.xml.sax.helpers.DefaultHandler</code> or
49  * <code>org.xml.sax.helpers.XMLFilterImpl</code> classes.
50  * You can subclass these if you need such behavior, or
51  * use the helper classes found here.</li>
52  *
53  * <li>The handlers need to be registered differently than core SAX2
54  * handlers.</li>
55  *
56  * </ul>
57  *
58  * <p>This package, SAX2-ext, is a standardized extension to SAX2.  It is
59  * designed both to allow SAX parsers to pass certain types of information
60  * to applications, and to serve as a simple model for other SAX2 parser
61  * extension packages.  Not all such extension packages should need to
62  * be recognized directly by parsers, however.
63  * As an example, most validation systems can be cleanly layered on top
64  * of parsers supporting the standardized SAX2 interfaces.
65  *
66  * @apiNote The SAX API, originally developed at
67  * <a href="http://www.saxproject.org">the SAX Project</a>,
68  * has been defined by Java SE since 1.4.
69  *
70  * @since 1.4
71  */
72
