  50     <h1><a id="file">File Format</a></h1>
  51     <p>
  52       Synth's file format (<a href="synth.dtd">dtd</a>)
  53       allows for specifying all the pieces
  54       necessary to create your own look and feel. A synth file is
  55       loaded by way of the <A
  56         HREF="../../../../../javax/swing/plaf/synth/SynthLookAndFeel.html#load(java.io.InputStream,java.lang.Class)">
  57       SynthLookAndFeel.load(InputStream, Class)</a> or
  58       <a href="../../../../../javax/swing/plaf/synth/SynthLookAndFeel.html#load(java.net.URL)">
  59         SynthLookAndFeel.load(URL)</a> methods.
  60       The following example uses the <code>load</code> method to configure
  61       a <code>SynthLookAndFeel</code> and sets it as the current look
  62       and feel:
  63     </p>
  64     <div class="example">
  65       <pre>
  66   SynthLookAndFeel laf = new SynthLookAndFeel();
  67   laf.load(MyClass.class.getResourceAsStream("laf.xml"), MyClass.class);
  68   UIManager.setLookAndFeel(laf);
  69       </pre>
  70     </div>
  71     <p>
  72       This example loads the look and feel from an input stream, using
  73       the specified class as the resource base to resolve paths.
  74       It is also possible to load a look and feel from an arbitrary URL
  75       as in the following example.
  76     </p>
  77     <div class="example">
  78       <pre>
  79   SynthLookAndFeel laf = new SynthLookAndFeel();
  80   laf.load(new URL("file:///C:/java/synth/laf/laf.xml"));
  81   UIManager.setLookAndFeel(laf);
  82       </pre>
  83     </div>
  84     <p>
  85       The method <a
  86         href="../../../../../javax/swing/plaf/synth/SynthLookAndFeel.html#load(java.net.URL)">
  87       SynthLookAndFeel.load(URL)</a> can be used, for instance, to load a look
  88       and feel from any of the following:
  89     </p>
  90     <ul>
  91       <li>File, e.g. <code>file:///C:/java/synth/laf/laf.xml</code></li>
  92       <li>Web server, e.g. <code>http://host/laf.xml</code></li>
  93       <li>JAR file, e.g.
  94         <code>jar:file:///C:/synth-laf.jar!/laf.xml</code></li>
  95       <li>Remote JAR file, e.g.
  96         <code>jar:http://host/synth-laf.jar!/laf.xml</code></li>
  97     </ul>
  98     <p>
  99       While the DTD for synth is specified, the parser is not validating.
 100       Parsing will fail only if a necessary attribute is not
 101       specified, or of the wrong type.

 123     <h2>The style element</h2>
 124 <div class="dtd-fragment">
 125 <pre class="dtd-fragment">
 126 &lt;!ELEMENT <a id="e.style">style</a> (<a href="#e.property">property</a> | <a href="#e.defaultsProperty">defaultsProperty</a> | <a href="#e.state">state</a> | <a href="#e.font">font</a> | <a href="#e.graphicsUtils">graphicsUtils</a> |
 127                  <a href="#e.insets">insets</a> | <a href="#e.painter">painter</a> | <a href="#e.imagePainter">imagePainter</a> | <a href="#e.opaque">opaque</a> | (<a href="#ee.beansPersistance">%beansPersistance;</a>) |
 128                  <a href="#e.imageIcon">imageIcon</a>)*>
 129 &lt;!ATTLIST style
 130           <a href="#style.id">id</a>              ID          #IMPLIED
 131           <a href="#style.clone">clone</a>           IDREF       #IMPLIED
 132 &gt;
 133 </pre>
 134 </div>
 135     <p><em>Attribute definitions</em></p>
 136     <dl>
 137       <dt><a id="style.id"><samp>id</samp></a></dt>
 138       <dd>Unique identifier for the style.</dd>
 139       <dt><a id="style.clone"><samp>clone</samp></a></dt>
 140       <dd>Identifier of a previously defined style that is copied
 141 	    and used for the new style. This provides a convenient
 142 	    mechanism for overriding only a portion of an existing
 143 	    style.</dd>
 144     </dl>
 145     <p>
 146       A style element corresponds to a
 147       <code>SynthStyle</code>, with the child elements specifying
 148       properties that apply to all states or state elements which
 149       contain properties specific to a particular state. The following
 150       example creates an opaque style with the id <code>button</code>,
 151       insets of 4, 4, 4, 4 and a font of Dialog 12.
 152     </p>
 153     <div class="example">
 154       <pre>
 155 &lt;style id="button">
 156   &lt;opaque value="true"/>
 157   &lt;insets top="4" left="4" right="4" bottom="4"/>
 158   &lt;font name="Dialog" size="12"/>
 159 &lt;/style>
 160       </pre>
 161     </div>
 162     <p>
 163       The following example creates a new style with an id of
 164       <code>clonedButton</code> that is a copy of the
 165       style with id <code>button</code> and has a font of Dialog,
 166       14. The resulting style will be opaque, have insets of 4, 4, 4,
 167       4 and a font of Dialog 14.
 168     </p>
 169     <div class="example">
 170       <pre>
 171 &lt;style id="clonedButton" clone="button">
 172   &lt;font name="Dialog" size="14"/>
 173 &lt;/style>
 174       </pre>
 175     </div>

 177     <h2>The state element</h2>
 178 <div class="dtd-fragment">
 179 <pre class="dtd-fragment">
 180 &lt;!ELEMENT <a id="e.state">state</a> (<a href="#e.color">color</a> | <a href="#e.font">font</a> | <a href="#e.painter">painter</a> | <a href="#e.imagePainter">imagePainter</a> | (<a href="#ee.beansPersistance">%beansPersistance;</a>) |
 181                  <a href="#e.property">property</a> |  <a href="#e.imageIcon">imageIcon</a>)*>
 182 &lt;!ATTLIST state
 183           <a href="#state.id">id</a>              ID             #IMPLIED
 184           <a href="#state.clone">clone</a>           IDREF          #IMPLIED
 185           <a href="#state.value">value</a>           CDATA          #IMPLIED
 186           <a href="#state.idref">idref</a>           IDREF          #IMPLIED
 187 &gt;
 188 </pre>
 189 </div>
 190     <p><em>Attribute definitions</em></p>
 191     <dl>
 192       <dt><a id="state.id"><samp>id</samp></a></dt>
 193       <dd>Unique identifier for the state.</dd>
 194       <dt><a id="state.clone"><samp>clone</samp></a></dt>
 195       <dd>Identifier of a previously defined state that is copied
 196 	and used for the new state.</dd>
 197       <dt><a id="state.value"><samp>value</samp></a></dt>
 198       <dd>Identifies the state of the Component the properties are to apply
 199 	to. This is a list of: ENABLED,
 200 	MOUSE_OVER, PRESSED, DISABLED, FOCUSED, SELECTED or
 201 	DEFAULT. Multiple states should
 202 	be separated by 'and.' If you do not specify a value, the
 203 	contents apply to all states.
 204       <dt><a id="state.idref"><samp>idref</samp></a></dt>
 205       <dd>Indicates this state should be the same as a previously
 206 	    defined state. This is useful for multiple styles that
 207 	    wish to share the same visual properties for a particular
 208 	    state.
 209       </dd>
 210     </dl>
 211     <p>
 212       The <a href="#e.state">state</a> element specifies the visual
 213       properties that are to be used for
 214       a particular state of a component. For example, you could
 215       specify the background color when the Component is enabled should look
 216       different than the background color when the component is
 217       disabled. Not all Components support all states. For example, a
 218       <code>Panel</code> only supports the states ENABLED and DISABLED.
 219     </p>
 220     <p>
 221       The following example creates a state with a red background that
 222       will be used when the component is in an a selected and pressed state:
 223     </p>
 224     <div class="example">
 225     <pre>
 226 &lt;state value="SELECTED AND PRESSED">
 227   &lt;color value="RED" type="BACKGROUND"/>
 228 &lt;/state>
 229     </pre>
 230 </div>
 231     <p>
 232       The state with the most individual matches will be
 233       chosen. For example, the following defines two states:
 234     </p>
 235     <div class="example">
 236       <pre>
 237 &lt;state value="SELECTED and PRESSED" id="one">
 238   &lt;color value="RED" type="BACKGROUND"/>
 239 &lt;/state>
 240 &lt;state value="SELECTED" id="two">
 241   &lt;color value="RED" type="BACKGROUND"/>
 242 &lt;/state>
 243     </pre>
 244 </div>
 245     <p>
 246       State <code>one</code> is used when the Component is SELECTED
 247       and PRESSED, and state <code>two</code> when the Component is
 248       SELECTED. If the state of the Component
 249       contains at least SELECTED and PRESSED, state <code>one</code> will be
 250       chosen, otherwise if the state is SELECTED, but not does not
 251       contain PRESSED, state <code>two</code> will be used.

 253     <h2>The font element</h2>
 254 <div class="dtd-fragment">
 255 <pre class="dtd-fragment">
 256 &lt;!ELEMENT <a id="e.font">font</a> EMPTY>
 257 &lt;!ATTLIST font
 258           <a href="#font.id">id</a>              ID             #IMPLIED
 259           <a href="#font.idref">clone</a>           IDREF          #IMPLIED
 260           <a href="#font.name">name</a>            CDATA          #IMPLIED
 261           <a href="#font.style">style</a>           CDATA          #IMPLIED
 262           <a href="#font.size">size</a>            CDATA          #IMPLIED
 263 &gt;
 264 </pre>
 265 </div>
 266     <p><em>Attribute definitions</em></p>
 267     <dl>
 268       <dt><a id="font.id"><samp>id</samp></a></dt>
 269       <dd>Unique identifier for the Font.</dd>
 270       <dt><a id="font.idref"><samp>idref</samp></a></dt>
 271       <dd>Identifier of a previously defined font.</dd>
 272       <dt><a id="font.name"><samp>name</samp></a></dt>
 273       <dd>Name of the font.
 274       <dt><a id="font.style"><samp>style</samp></a></dt>
 275       <dd>Style of the font. This is a list of the styles defined by
 276 	Font separated by spaces: PLAIN, BOLD or ITALIC. If
 277 	unspecified PLAIN is used.
 278       </dd>
 279       <dt><a id="font.size"><samp>size</samp></a></dt>
 280       <dd>Size of the font, in pixels</dd>
 281     </dl>
 282     <p>
 283       Defines the font for the current <a href="#e.state">state</a>,
 284       or <a href="#e.style">style</a>. You must
 285       specify either an <a href="#font.idref">idref</a> or a
 286       <a href="#font.name">name</a> and <a href="#font.size">size</a>.
 287     </p>
 288     <p>
 289       The following example creates a style with a Font of
 290       Dialog 12 Bold.
 291     </p>
 292     <div class="example">
 293       <pre>
 294 &lt;style id="test">
 295   &lt;font name="DIALOG" size="12" style="BOLD"/>
 296 &lt;/style>
 297     </pre>
 298 </div>
 299     <p>
 300       The following example creates a style with a font of
 301       Dialog 12 Bold that will be used if the component is ENABLED,
 302       otherwise Dialog 12 Italic will be used.
 303     </p>
 304     <div class="example">
 305       <pre>
 306 &lt;style id="test">
 307   &lt;font name="DIALOG" size="12" style="ITALIC"/>
 308   &lt;state value="ENABLED">
 309     &lt;font name="DIALOG" size="12" style="BOLD"/>
 310   &lt;/state>
 311 &lt;/style>
 312     </pre>
 313 </div>
 314     <p>
 315       While you can supply a different font per state, in general
 316       widgets will NOT revalidate when the state changes, so that you
 317       may run into sizing problems if you try to use a font with a
 318       significantly different size for different states.
 319     </p>

 322     <h2>The color element</h2>
 323 <div class="dtd-fragment">
 324 <pre class="dtd-fragment">
 325 &lt;!ELEMENT <a id="e.color">color</a> EMPTY>
 326 &lt;!ATTLIST color
 327           <a href="#color.id">id</a>              ID             #IMPLIED
 328           <a href="#color.idref">idref</a>           IDREF          #IMPLIED
 329           <a href="#color.type">type</a>            CDATA          #IMPLIED
 330           <a href="#color.value">value</a>           CDATA          #IMPLIED
 331 &gt;
 332 </pre>
 333 </div>
 334     <p><em>Attribute definitions</em></p>
 335     <dl>
 336       <dt><a id="color.id"><samp>id</samp></a></dt>
 337       <dd>Unique identifier for the color.</dd>
 338       <dt><a id="color.idref"><samp>idref</samp></a></dt>
 339       <dd>Identifier of a previously defined color.</dd>
 340       <dt><a id="color.type"><samp>type</samp></a></dt>
 341       <dd>Describes where this color should be used. This is
 342 	    typically one of the constants defined by ColorType:
 343 	    FOREGROUND, BACKGROUND, TEXT_FOREGROUND, TEXT_BACKGROUND
 344 	    or FOCUS. Alternatively you can specify the complete path
 345 	    to a class and field, for example
 346 	    javax.swing.plaf.synth.ColorType.FOREGROUND, this
 347 	    is useful for subclasses of synth that define additional
 348 	    color types.
 349       </dd>
 350       <dt><a id="color.value"><samp>value</samp></a></dt>
 351       <dd>
 352         Value for the color. This accepts the following forms.
 353         <ul>
 354           <li>The name of a constant in the <code>Color</code> class,
 355             for example <code> RED</code>.
 356           <li>A hex value of the form <code>#RRGGBB</code> where
 357             <code>RR</code> gives the red component, <code>GG</code>
 358             the green component and <code>BB</code> the blue
 359             component. You need not specify all color components. For
 360             example, <code>#123</code> is equivalent to <code>#000123</code>.
 361           <li>A hex value of the form <code>#ARRGGBB</code> or
 362             <code>#AARRGGBB</code>. This is useful for alpha values
 363             other than <code>0xFF</code>. The form
 364             <code>#ARRGGBB</code> is equivalent to
 365             <code>#0ARRGGBB</code>.
 366         </ul>
 367       </dd>
 368     </dl>
 369     <p>
 370       <a href="#e.color">Color</a> defines a color and what portion of
 371       the Component it should be applied to. The following example
 372       will use a background color of RED when the component is
 373       enabled.
 374     </p>
 375     <div class="example">
 376       <pre>
 377 &lt;state value="ENABLED">
 378   &lt;color value="RED" type="BACKGROUND"/>
 379 &lt;/state>
 380     </pre>
 381 </div>
 382     <p>
 383       The following example will have a red background when the
 384       Component is enabled, otherwise blue.
 385     </p>
 386     <div class="example">
 387       <pre>
 388 &lt;style id="test">
 389   &lt;state value="ENABLED">
 390     &lt;color value="RED" type="BACKGROUND"/>
 391   &lt;/state>
 392   &lt;state>
 393     &lt;color value="#00FF00" type="BACKGROUND"/>
 394   &lt;/state>
 395 &lt;/style>
 396     </pre>
 397 </div>

 399     <h2>The property element</h2>
 400 <div class="dtd-fragment">
 401 <pre class="dtd-fragment">
 402 &lt;!ELEMENT <a id="e.property">property</a> EMPTY>
 403 &lt;!ATTLIST property
 404           <a href="#property.key">key</a>               CDATA                                             #REQUIRED
 405           <a href="#property.type">type</a>              (idref|boolean|dimension|insets|integer|string)          "idref"
 406           <a href="#property.value">value</a>             CDATA                                             #REQUIRED
 407 &gt;
 408 </pre>
 409 </div>
 410     <p><em>Attribute definitions</em></p>
 411     <dl>
 412       <dt><a id="property.key"><samp>key</samp></a></dt>
 413       <dd>Name of the property.</dd>
 414       <dt><a id="property.type"><samp>type</samp></a></dt>
 415       <dd>Indicates the type of the property.</dd>
 416       <dt><a id="property.value"><samp>value</samp></a></dt>
 417       <dd>Value for the property. For boolean properties this will be
 418 	be true or false, for integer properties this will be a
 419 	valid integer, for dimensions this will be the width and
 420 	height separated by a space, for insets properties this will
 421 	be the top, left, bottom and right separated by a space and
 422 	for idref properties this will be the unique id of a
 423 	previously defined object.</dd>
 424     </dl>
 425     <p>
 426       <a href="#e.property">Property</a> elements are used to add key value pairs to a
 427       <code>SynthStyle</code> that can be accessed by way of the
 428       <code>get</code> method. Many <code>Component</code>s use the
 429       key value pairs for configuring their visual appearance. Refer to
 430       <a href="componentProperties.html">property table</a> for a list of the
 431       properties each <code>Component</code> supports. The following
 432       creates the properties
 433       <code>ScrollBar.allowsAbsolutePositioning</code>,
 434       <code>OptionPane.minimumSize</code>,
 435       <code>ScrollPane.viewportBorderInsets</code>,
 436       <code>Tree.rowHeight</code> and <code>foreground</code> with the
 437       values false, a dimensions of 262x90, an insets of 5, 5, 5, 5,
 438       the integer 20 and an instance of the
 439       class ArrowButtonPainter.
 440     </p>
 441     <div class="example">
 442       <pre>
 443 &lt;style id="test">
 444   &lt;property key="ScrollBar.allowsAbsolutePositioning" type="boolean" value="false"/>
 445   &lt;property key="OptionPane.minimumSize" type="dimension" value="262 90"/>
 446   &lt;property key="ScrollPane.viewportBorderInsets" type="insets" value="5 5 5 5"/>
 447   &lt;property key="Tree.rowHeight" type="integer" value="20"/>
 448   &lt;object class="ArrowButtonPainter" id="ArrowButtonPainter"/>
 449   &lt;property key="foreground" type="idref" value="ArrowButtonPainter"/>
 450 &lt;/style>
 451     </pre>
 452 </div>
 453     <p>
 454       You can also specify properties that are to apply to specific
 455       states.  Whether or not the property is accessed for each state
 456       depends upon how the property is used.  For example, the
 457       following specifies a default icon and an icon to use while the
 458       mouse is over the component.
 459     </p>
 460     <div class="example">
 461       <pre>
 462 &lt;style id="test">
 463   &lt;imageIcon id="defaultIcon" path="resources/myImage.png"/>
 464   &lt;property key="RadioButton.icon" value="defaultIcon"/>
 465   &lt;state value="MOUSE_OVER">
 466     &lt;imageIcon id="mouseOverIcon" path="resources/myMouseOverImage.png"/>
 467     &lt;property key="RadioButton.icon" value="mouseOverIcon"/>
 468   &lt;/state>
 469 &lt;/style>
 470     </pre>
 471 </div>
 472 <div class="dtd-fragment">
 473 <pre class="dtd-fragment">
 474 &lt;!ELEMENT <a id="e.defaultsProperty">defaultsProperty</a> EMPTY>
 475 &lt;!ATTLIST defaultsProperty
 476           <a href="#defaultsProperty.key">key</a>               CDATA                                             #REQUIRED
 477           <a href="#defaultsProperty.type">type</a>              (idref|boolean|dimension|insets|integer|string)   "idref"
 478           <a href="#defaultsProperty.value">value</a>             CDATA                                             #REQUIRED
 479 &gt;
 480 </pre>
 481 </div>
 482     <p><em>Attribute definitions</em></p>
 483     <dl>
 484       <dt><a id="defaultsProperty.key"><samp>key</samp></a></dt>
 485       <dd>Name of the property.</dd>
 486       <dt><a id="defaultsProperty.type"><samp>type</samp></a></dt>
 487       <dd>Indicates the type of the property.</dd>
 488       <dt><a id="defaultsProperty.value"><samp>value</samp></a></dt>
 489       <dd>Value for the property. For boolean properties this will be
 490 	true or false, for integer properties this will be a
 491 	valid integer, for dimensions this will be the width and
 492 	height separated by a space, for insets properties this will
 493 	be the top, left, bottom and right separated by a space and
 494 	for idref properties this will be the unique id of a
 495 	previously defined object.</dd>
 496     </dl>
 497     <p>
 498       <a href="#e.defaultsProperty">DefaultsProperty</a> elements are
 499       used to define properties that will be placed in the
 500       <code>UIDefaults</code> table that <code>SynthLookAndFeel</code>
 501       supplies to the <code>UIManager</code>. The following assigns the
 502       the Color red to the value Table.focusCellForeground.
 503     </p>
 504     <div class="example">
 505       <pre>
 506 &lt;style id="test">
 507   &lt;object class="javax.swing.plaf.ColorUIResource" id="color">
 508     &lt;int>255&lt;/int>
 509     &lt;int>0&lt;/int>
 510     &lt;int>0&lt;/int>
 511   &lt;/object>
 512   &lt;defaultsProperty key="Table.focusCellForeground" type="idref" value="color"/>
 513 &lt;/style>
 514     </pre>
 515 </div>
 516     <p>
 517       This value could then be asked by way of
 518       <code>UIManager.get("Table.focusCellForeground")</code>.
 519     </p>

 521     <h2>The graphicsUtils element</h2>
 522 <div class="dtd-fragment">
 523 <pre class="dtd-fragment">
 524 &lt;!ELEMENT <a id="e.graphicsUtils">graphicsUtils</a> EMPTY>
 525 &lt;!ATTLIST graphicsUtils
 526           <a href="#graphicsUtils.idref">idref</a>           IDREF             #REQUIRED
 527 &gt;
 528 </pre>
 529 </div>
 530     <p><em>Attribute definitions</em></p>
 531     <dl>
 532       <dt><a id="graphicsUtils.idref"><samp>idref</samp></a></dt>
 533       <dd>Identifer of a previously defined SynthGraphicsUtils object
 534 	that is to be used as the SynthGraphicsUtils for the current
 535 	<a href="#e.style">style</a>.</dd>
 536     </dl>
 537     <p>
 538       <a href="#e.graphicsUtils">GraphicsUtils</a> elements are
 539       used to define the SynthGraphicsUtils that the current
 540       <a href="#e.style">style</a> will use. The following example
 541       creates a style with an instance of CustomGraphicsUtils for the
 542       SynthGraphicsUtils.
 543     </p>
 544     <div class="example">

 597     <h2>The bind element</h2>
 598 <div class="dtd-fragment">
 599 <pre class="dtd-fragment">
 600 &lt;!ELEMENT <a id="e.bind">bind</a> EMPTY>
 601 &lt;!ATTLIST bind
 602           <a href="#bind.style">style</a>        IDREF             #REQUIRED
 603           <a href="#bind.type">type</a>         (name|region)     #REQUIRED
 604           <a href="#bind.key">key</a>          CDATA             #REQUIRED
 605 &gt;
 606 </pre>
 607 </div>
 608     <p><em>Attribute definitions</em></p>
 609     <dl>
 610       <dt><a id="bind.style"><samp>style</samp></a></dt>
 611       <dd>Unique identifier of a previously defined style.</dd>
 612       <dt><a id="bind.type"><samp>type</samp></a></dt>
 613       <dd>One of name or region. For type name component.getName() is used,
 614 	otherwise the name of the Region is used.</dd>
 615       <dt><a id="bind.key"><samp>key</samp></a></dt>
 616       <dd>Regular expression applied to the name of the Component, or the
 617 	name of the Region, depending upon the value of
 618 	<a href="#bind.type">type</a>.</dd>
 619     </dl>
 620     <p>
 621       <a href="#e.bind">Bind</a> elements specify which Regions a style
 622       is to be used for. The following example applies the
 623       <a href="#e.style">style</a> test to any Component whose name
 624       starts with <code>test</code>.
 625     </p>
 626     <div class="example">
 627       <pre>
 628 &lt;style id="test">
 629   &lt;insets top="1" bottom="2" left="3"/>
 630 &lt;/style>
 631 &lt;bind style="test" type="name" key="test.*"/>
 632     </pre>
 633 </div>
 634     <p>
 635       Numerous styles may apply to a region, in which case each of
 636       the matching styles is merged into a resulting style that is used.
 637       Precedence is given to styles defined later in the file. For example,
 638       the following defines two styles, a and b. Style a is applied to any
 639       component with a name starting with test, and style b is used for
 640       button regions.
 641     </p>
 642     <div class="example">
 643       <pre>
 644 &lt;style id="a">
 645   &lt;font name="DIALOG" size="12" style="ITALIC"/>
 646   &lt;insets top="1" bottom="2" left="3"/>
 647 &lt;/style>
 648 &lt;bind style="a" type="name" key="test.*"/>
 649 &lt;style id="b">
 650   &lt;font name="DIALOG" size="12" style="BOLD"/>
 651 &lt;/style>
 652 &lt;bind style="b" type="region" key="button"/>
 653     </pre>
 654 </div>
 655     <p>
 656       For a button with the name test this is equivalent to:
 657     </p>
 658     <div class="example">
 659       <pre>
 660 &lt;style>
 661   &lt;font name="DIALOG" size="12" style="BOLD"/>
 662   &lt;insets top="1" bottom="2" left="3"/>
 663 &lt;/style>
 664     </pre>
 665 </div>
 666     <p>
 667       Merging happens for states of a style as well.
 668     </p>
 669     <div class="example">
 670       <pre>
 671 &lt;style id="a">
 672   &lt;font name="DIALOG" size="12" style="ITALIC"/>
 673   &lt;insets top="1" bottom="2" left="3"/>
 674   &lt;state value="ENABLED">
 675     &lt;object id="customPainter" class="CustomPainter"/>
 676     &lt;painter idref="customPainter"/>
 677   &lt;/state>
 678 &lt;/style>
 679 &lt;bind style="a" type="name" key="test.*"/>
 680 &lt;style id="b">
 681   &lt;font name="DIALOG" size="12" style="BOLD"/>
 682   &lt;state value="ENABLED">
 683     &lt;font name="Lucida" size="12" style="ITALIC"/>
 684   &lt;/state>
 685 &lt;/style>
 686 &lt;bind style="b" type="region" key="button"/>
 687     </pre>
 688 </div>
 689     <p>
 690       For a button with the name test this is equivalent to:
 691     </p>
 692     <div class="example">
 693       <pre>
 694 &lt;style>
 695   &lt;font name="DIALOG" size="12" style="BOLD"/>
 696   &lt;insets top="1" bottom="2" left="3"/>
 697   &lt;state value="ENABLED">
 698     &lt;object id="customPainter" class="CustomPainter"/>
 699     &lt;painter idref="customPainter"/>
 700     &lt;font name="Lucida" size="12" style="ITALIC"/>
 701   &lt;/state>
 702 &lt;/style>
 703     </pre>

 706     <h2>The painter element</h2>
 707 <div class="dtd-fragment">
 708 <pre class="dtd-fragment">
 709 &lt;!ELEMENT <a id="e.painter">painter</a> EMPTY>
 710 &lt;!ATTLIST painter
 711           <a href="#painter.idref">idref</a>                 IDREF          #IMPLIED
 712           <a href="#painter.method">method</a>                CDATA          #IMPLIED
 713           <a href="#painter.direction">direction</a>             (north|south|east|west|top|left|bottom|right|horizontal|vertical|horizontal_split|vertical_split)          #IMPLIED
 714 &gt;
 715 </pre>
 716 </div>
 717     <p><em>Attribute definitions</em></p>
 718     <dl>
 719       <dt><a id="painter.idref"><samp>idref</samp></a></dt>
 720       <dd>Identifier of a previously defined SynthPainter.</dd>
 721       <dt><a id="painter.method"><samp>method</samp></a></dt>
 722       <dd>Identifies the SynthPainter method this is to be used for. The name
 723 	corresponds to the method name of a paint method in SynthPainter
 724 	with the paint prefix dropped, the remainder is case
 725 	insensitive (using the latin1 case folding rules).
 726 	For example SynthPainter.paintButtonBackground is identified by
 727 	'buttonBackground' or 'buttonbackground'. If this is
 728 	not specified the painter is used for all methods that don't have a
 729 	a specific painter for them.</dd>
 730       <dt><a id="painter.direction"><samp>direction</samp></a></dt>
 731       <dd>Identifies the direction, or orientation, this painter is to be
 732 	used for. This is only useful for the SynthPainter methods that take
 733 	a direction or orientation. If this is not specified the painter is
 734 	used for all directions.</dd>
 735     </dl>
 736     <p>
 737       <a href="#e.painter">Painter</a> defines a SynthPainter for the current
 738       style or the state of the current style. The following example
 739       binds an instance of the class <code>MyPainter</code>
 740       which must be a <code>SynthPainter</code> to the style <code>test</code>.
 741     </p>
 742     <div class="example">
 743       <pre>
 744 &lt;style id="test">
 745   &lt;object class="MyPainter" id="MyPainter"/>
 746   &lt;painter idref="MyPainter"/>
 747 &lt;/style>
 748     </pre>
 749 </div>
 750     <p>
 751       The painter that is used for a particular method and state is determined
 752       as follows:
 753     <ol>
 754       <li>Painter specified for the current state, method and direction.
 755       <li>Painter specified for the current state and method.
 756       <li>Painter specified for the current state.
 757       <li>Painter specified for the style, method and direction.
 758       <li>Painter specified for the style and method.
 759       <li>Painter specified for the style.
 760     </ol>
 761     <p>
 762       Consider the following:
 763     </p>
 764     <div class="example">
 765       <pre>
 766 &lt;style id="test">
 767   &lt;painter idref="fallbackPainter"/>
 768   &lt;painter idref="styleButtonBackgroundPainter" method="buttonBackground"/>
 769   &lt;state value="SELECTED">
 770     &lt;painter idref="stateFallbackPainter"/>
 771     &lt;painter idref="stateButtonBackgroundPainter" method="buttonBackground"/>
 772   &lt;/state>
 773 &lt;/style>
 774       </pre>
 775     </div>
 776     <p>
 777       The following outlines which painter will be used for what
 778       SynthPainter method:
 779     </p>
 780     <table border=1>
 781      <caption>Painters for SynthPainter methods</caption>
 782       <thead>
 783       <tr>
 784         <th scope="col">Index
 785         <th scope="col">State
 786         <th scope="col">Method
 787         <th scope="col">Painter
 788       </thead>
 789       <tbody>
 790 	    <tr>
 791         <th scope="row">1
 792         <td>SELECTED
 793         <td>paintButtonBackground<td>stateButtonBackgroundPainter
 794 	    <tr>
 795         <th scope="row">2
 796         <td>SELECTED
 797         <td>Anything but paintButtonBackground<td>stateFallbackPainter
 798 	    <tr>
 799         <th scope="row">3
 800         <td>Anything but SELECTED
 801         <td>paintButtonBackground<td>styleButtonBackgroundPainter
 802 	    <tr>
 803         <th scope="row">4
 804         <td>Anything but SELECTED
 805         <td>Anything but paintButtonBackground<td>fallbackPainter
 806       </tbody>
 807     </table>
 808 
 809     <p>
 810       When several identical painters are declared, they are aggregated into
 811       a single one. Two painters are identical if their <em>direction</em> and
 812       <em>method</em> attributes values are equal. Consider the following:
 813     </p>
 814 
 815 	<div class="example">
 816 	  <pre>
 817 &lt;style id="panelStyle"&gt;
 818   &lt;imagePainter method="panelBackground" path="red.png" /&gt;
 819   &lt;imagePainter method="panelBackground" path="green.png" /&gt;
 820   &lt;imagePainter method="panelBackground" path="blue.png" /&gt;
 821 &lt;/style&gt;
 822 	  </pre>
 823 	</div>
 824 
 825 	<p>
 826 	  These three painters are identical for they use the same method and the
 827 	  same direction (all directions by default). Synth aggregates these painters
 828 	  to create a single one that will paint its children painters in the order
 829 	  of declaration. Hence, Synth will first paint the red picture, then the green
 830 	  one and finally the blue one. Each child painter can be seen as a layer of
 831 	  the aggregate painter.
 832 	</p>
 833 	<p>
 834 	  Painter aggregation, or multi-layering, is very useful to reuse elements.
 835 	  Imagine you want to use an highlight effect on buttons and on selected
 836 	  menu items. With painter aggregation, you just need to create a separate
 837 	  highlighting painter instead of having buttons and menu items painters
 838 	  handle it.
 839 	</p>

 841     <h2>The imagePainter element</h2>
 842 <div class="dtd-fragment">
 843 <pre class="dtd-fragment">
 844 &lt;!ELEMENT <a id="e.imagePainter">imagePainter</a> EMPTY>
 845 &lt;!ATTLIST imagePainter
 846           <a href="#imagePainter.id">id</a>                    ID             #IMPLIED
 847           <a href="#imagePainter.method">method</a>                CDATA          #IMPLIED
 848           <a href="#imagePainter.direction">direction</a>             (north|south|east|west|top|left|bottom|right|horizontal|vertical|horizontal_split|vertical_split)          #IMPLIED
 849           <a href="#imagePainter.path">path</a>                  CDATA          #REQUIRED
 850           <a href="#imagePainter.sourceInsets">sourceInsets</a>          CDATA          #IMPLIED
 851           <a href="#imagePainter.destinationInsets">destinationInsets</a>     CDATA          #IMPLIED
 852           <a href="#imagePainter.painterCenter">paintCenter</a>           (true|false)   "true"
 853           <a href="#imagePainter.stretch">stretch</a>               (true|false)   "true"
 854           <a href="#imagePainter.center">center</a>                (true|false)   "false"
 855 &gt;
 856 </pre>
 857 </div>
 858     <p><em>Attribute definitions</em></p>
 859     <dl>
 860       <dt><a id="imagePainter.id"><samp>id</samp></a></dt>
 861       <dd>Unique identifier for the imagePainter.</dd>
 862       <dt><a id="imagePainter.method"><samp>method</samp></a></dt>
 863       <dd>Identifies the SynthPainter method this is to be used for. The name
 864 	corresponds to the method name of a paint method in SynthPainter
 865 	with the paint prefix dropped, the remainder is case
 866 	insensitive (using the latin1 case folding rules).
 867 	For example SynthPainter.paintButtonBackground is identified by
 868 	'buttonBackground' or 'buttonbackground'. If this is
 869 	not specified the painter is used for all methods that don't have a
 870 	a specific painter for them.</dd>
 871       <dt><a id="imagePainter.direction"><samp>direction</samp></a></dt>
 872       <dd>Identifies the direction, or orientation, this image is to be
 873 	used for. This is only useful for the SynthPainter methods that take
 874 	a direction or orientation. If this is not specified the image is
 875 	used for all directions.</dd>
 876       <dt><a id="imagePainter.path"><samp>path</samp></a></dt>
 877       <dd>Path to the image. Path to the image.  If SynthLookAndFeel.load is
 878     passed a Class this will use the Class method getResource (with with the
 879     Class suplied to the load method). If load is passed a URL this will use the
 880     URL constructor URL(context, path) to resolve the path.</dd>
 881       <dt><a id="imagePainter.sourceInsets"><samp>sourceInsets</samp></a></dt>
 882       <dd>Insets on the source image. This is top, left, bottom, right with
 883 	each component separated by a space.</dd>
 884       <dt><a id="imagePainter.destinationInsets"><samp>destinationInsets</samp></a></dt>
 885       <dd>Insets of the destination image. This is top, left, bottom, right with
 886 	each component separated by a space. If not specified the
 887 	<a href="#imagePainter.sourceInsets">sourceInsets</a> are used.</dd>
 888       <dt><a id="imagePainter.painterCenter"><samp>paintCenter</samp></a></dt>
 889       <dd>Whether or not the center of the image should be drawn.</dd>
 890       <dt><a id="imagePainter.stretch"><samp>stretch</samp></a></dt>
 891       <dd>Whether or not the north, south, east and west components of the
 892 	resulting image should be scaled or tiled.</dd>
 893       <dt><a id="imagePainter.center"><samp>center</samp></a></dt>
 894       <dd>Whether or not the image is centered.</dd>
 895     </dl>
 896     <p>
 897       The <a href="#e.imagePainter">ImagePainter</a> element defines a
 898       painter for the current style or state that will render using
 899       the specified image.  ImagePainter offers two distinct rendering
 900       modes.  The first mode is used to center an image in the space
 901       provided.  This is
 902       commonly used in rendering decorations on top of a widget, for
 903       example, to specify an arrow for a scroll button use the center
 904       mode. The following example illustrates this:
 905     <div class="example">
 906       <pre>
 907 &lt;style id="test">
 908   &lt;imagePainter path="resources/myImage.png" center="true"/>
 909 &lt;/style>
 910     </pre>
 911 </div>
 912     <p>
 913       The second mode is used in scaling an image to fit in the
 914       provided space.  In this mode sourceInsets is used to specify a
 915       border around an image where the north, south, east and west
 916       edges of the border are either stretched or tiled (stretch
 917       attribute), the four corners of the border drawn in place, and
 918       the center is stretched.  In this mode you must specify
 919       sourceInsets.  The following example illustrates an
 920       ImagePainter that is using the image MyImage.png and insets of 2
 921       all the way around:
 922     </p>
 923     <div class="example">
 924       <pre>
 925 &lt;style id="test">
 926   &lt;imagePainter path="resources/myImage.png"
 927                 sourceInsets="2 2 2 2"/>
 928 &lt;/style>
 929     </pre>
 930 </div>
 931     <p>
 932       Refer to the description of the <a href="#e.painter">painter</a>
 933       element for details as to the precedence in choosing a painter and to
 934       understand how identical painters are handled.
