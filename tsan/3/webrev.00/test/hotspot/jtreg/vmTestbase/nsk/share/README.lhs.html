 22 
 23 This directory contains source files of NSK tests framework
 24 shared between all NSK tests.
 25 
 26 Files located directly in this directory provide general support
 27 for all tests.
 28 
 29 Files in the subdirectories provide specific support for tests of
 30 particular subsuites.
 31 
 32 ---------------------------------------------------------------------------------
 33 
 34 Short description of files:
 35 
 36     common exceptions:
 37         Failure,java, TestBug.java, Oddity.java
 38     common constants:
 39         Consts.java
 40     parsing command line arguments:
 41         ArgumentPareser.java
 42     output of errors and messages:
 43         Log.java
 44     process running:
 45         LocalProcess.java, IORedirector.java
 46     class loading/unloading:
 47         DummyClassLoader.java, ZipClassLoader.java,
 48         CustomClassLoader.java, ClassUnloder.java
 49     objects finalization:
 50         Finalizable.java, FinalizableObject.java, Finalizer.java
 51     threads synchronization:
 52         Wicket.java
 53     text processing:
 54         Grep.java, Paragrep.java
 55     timeouts handling:
 56         Harakiri.java, TimeoutHandler.java
 57     tree structures support:
 58         Denotation.java, TreeNodesDenotation.java
 59     RAS mode support:
 60         RASagent.java, JVMTIagent.c
 61     JVMDI tests support:
 62         JVMDITools.h, JVMDITools.c
 63 
 64 Short description of subdirectories:
 65 
 66     Alien       - support for accessing external tests (JCK)
 67     native      - support for native part of NSK tests
 68     jni         - support for JNI tests and accessing JNI API
 69     jvmti       - support for JVMTI tests and accessing JVMTI API
 70     jpda        - support for two-VMs JPDA tests
 71     jdwp        - support for JDWP tests and accessing JDWP API
 72     jdi         - support for JDI tests and accesing JDI API
 73     jdb         - support for JDB tests and accessing JDB tool
 74     monitoring  - support for monitoring tests and accessing Java Monitoring&Management API
 75     sysdict     - support for System Dictionary tests
 76     gc          - support for GC tests
 77     regression  - support for regression tests for known bugs
 78     split_verifier - support for Split Verifier tests
 79 
 80 For more detailed description see README files in subdirectories.

 84 Naming conventions
 85 
 86 Classes:
 87 
 88     All shared classes are groupped into packages to prevent
 89     name collision.
 90 
 91     All classes exported directly from this directory are
 92     of package:
 93 
 94         nsk.share
 95 
 96     All classes exported from subdirectories are of particular
 97     subpackage, e.g.:
 98 
 99         nsk.share.jpda
100         nsk.share.jdwp
101         nsk.share.jdi
102         nsk.share.jdb
103         nsk.share.sysdict
104 
105 Native functions and macroses:
106 
107     Most native functions have special prefix to prevent linking collisions.
108     Most macroses also have special prefix and are wrote in upper register.
109 
110     Here is typical naming scheme used for native functions and macroses:
111 
112         share/native
113             functions: nsk_*
114             macroses:  NSK_*
115 
116         share/jni
117             functions: nsk_jni_*
118             macroses:  NSK_JNI_*
119 
120         share/jvmti
121             functions: nsk_jvmti_*
122             macroses:  NSK_JVMTI_*
123 
124     However, some native functions and macroses do not follow this scheme,
125     in order to preserve compatibility with old tests.
