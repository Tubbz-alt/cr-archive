 92 <h1 class="title">How This API Document Is Organized (custom help file)</h1>
 93 <div class="subTitle">This API (Application Programming Interface) document has pages corresponding to the items in the navigation bar, described as follows.</div>
 94 </div>
 95 <div class="contentContainer">
 96 <ul class="blockList">
 97 <li class="blockList">
 98 <h2>Package</h2>
 99 <p>Each package has a page that contains a list of its classes and interfaces, with a summary for each. This page can contain six categories:</p>
100 <ul>
101 <li>Interfaces (italic)</li>
102 <li>Classes</li>
103 <li>Enums</li>
104 <li>Exceptions</li>
105 <li>Errors</li>
106 <li>Annotation Types</li>
107 </ul>
108 </li>
109 <li class="blockList">
110 <h2>Class/Interface</h2>
111 <p>Each class, interface, nested class and nested interface has its own separate page. Each of these pages has three sections consisting of a class/interface description, summary tables, and detailed member descriptions:</p>
112 <ul>
113 <li>Class inheritance diagram</li>
114 <li>Direct Subclasses</li>
115 <li>All Known Subinterfaces</li>
116 <li>All Known Implementing Classes</li>
117 <li>Class/interface declaration</li>
118 <li>Class/interface description</li>
119 </ul>
120 <ul>
121 <li>Nested Class Summary</li>
122 <li>Field Summary</li>
123 <li>Constructor Summary</li>
124 <li>Method Summary</li>
125 </ul>
126 <ul>
127 <li>Field Detail</li>
128 <li>Constructor Detail</li>
129 <li>Method Detail</li>
130 </ul>
131 <p>Each summary entry contains the first sentence from the detailed description for that item. The summary entries are alphabetical, while the detailed descriptions are in the order they appear in the source code. This preserves the logical groupings established by the programmer.</p>
132 </li>
133 <li class="blockList">
134 <h2>Annotation Type</h2>
135 <p>Each annotation type has its own separate page with the following sections:</p>
136 <ul>
137 <li>Annotation Type declaration</li>
138 <li>Annotation Type description</li>
139 <li>Required Element Summary</li>
140 <li>Optional Element Summary</li>
141 <li>Element Detail</li>
142 </ul>
143 </li>
144 <li class="blockList">
145 <h2>Enum</h2>
146 <p>Each enum has its own separate page with the following sections:</p>
147 <ul>
148 <li>Enum declaration</li>
149 <li>Enum description</li>
150 <li>Enum Constant Summary</li>
151 <li>Enum Constant Detail</li>
152 </ul>
153 </li>
154 <li class="blockList">
155 <h2>Tree (Class Hierarchy)</h2>
156 <p>There is a <a href="overview-tree.html">Class Hierarchy</a> page for all packages, plus a hierarchy for each package. Each hierarchy page contains a list of classes and a list of interfaces. The classes are organized by inheritance structure starting with <code>java.lang.Object</code>. The interfaces do not inherit from <code>java.lang.Object</code>.</p>
157 <ul>
158 <li>When viewing the Overview page, clicking on "Tree" displays the hierarchy for all packages.</li>
159 <li>When viewing a particular package, class or interface page, clicking "Tree" displays the hierarchy for only that package.</li>
160 </ul>
161 </li>
162 <li class="blockList">
163 <h2>Deprecated API</h2>
164 <p>The <a href="deprecated-list.html">Deprecated API</a> page lists all of the API that have been deprecated. A deprecated API is not recommended for use, generally due to improvements, and a replacement API is usually given. Deprecated APIs may be removed in future implementations.</p>
165 </li>
166 <li class="blockList">
167 <h2>Index</h2>
168 <p>The <a href="index-all.html">Index</a> contains an alphabetic list of all classes, interfaces, constructors, methods, and fields.</p>
169 </li>
170 <li class="blockList">
171 <h2>Prev/Next</h2>
172 <p>These links take you to the next or previous class, interface, package, or related page.</p>
173 </li>
174 <li class="blockList">
175 <h2>Frames/No Frames</h2>
176 <p>These links show and hide the HTML frames.  All pages are available with or without frames.</p>
177 </li>
178 <li class="blockList">
179 <h2>All Classes</h2>
180 <p>The <a href="allclasses-noframe.html">All Classes</a> link shows all classes and interfaces except non-static nested types.</p>
181 </li>
182 <li class="blockList">
183 <h2>Serialized Form</h2>
184 <p>Each serializable or externalizable class has a description of its serialization fields and methods. This information is of interest to re-implementors, not to developers using the API. While there is no link in the navigation bar, you can get to this information by going to any serialized class and clicking "Serialized Form" in the "See also" section of the class description.</p>
185 </li>
186 <li class="blockList">
187 <h2>Constant Field Values</h2>
188 <p>The <a href="constant-values.html">Constant Field Values</a> page lists the static final fields and their values.</p>
189 </li>
190 </ul>
