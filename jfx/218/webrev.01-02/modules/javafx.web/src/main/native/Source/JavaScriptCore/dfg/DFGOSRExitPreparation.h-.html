35 // Make sure all code on our inline stack is JIT compiled. This is necessary since
36 // we may opt to inline a code block even before it had ever been compiled by the
37 // JIT, but our OSR exit infrastructure currently only works if the target of the
38 // OSR exit is JIT code. This could be changed since there is nothing particularly
39 // hard about doing an OSR exit into the interpreter, but for now this seems to make
40 // sense in that if we're OSR exiting from inlined code of a DFG code block, then
41 // probably it's a good sign that the thing we're exiting into is hot. Even more
42 // interestingly, since the code was inlined, it may never otherwise get JIT
43 // compiled since the act of inlining it may ensure that it otherwise never runs.
44 void prepareCodeOriginForOSRExit(ExecState*, CodeOrigin);
45 
46 } } // namespace JSC::DFG
